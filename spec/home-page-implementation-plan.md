# Home Page Implementation Plan (Revised)

## Context

The indvaltech-web project (Astro 6 + Tailwind CSS 4 + React) has a home page that was initially scaffolded but does not match the Stitch design ("Home — Indvaltech Revamp"). This plan corrects all visual and structural deviations identified by comparing the current code against the actual Stitch HTML output.

**Critical design failures in the current build:**
- Navbar background is dark navy (inverted from the Stitch design, which uses light glassmorphism)
- Nav item labels use wrong font (Inter instead of Space Grotesk), no uppercase, no tracking, wrong hover color
- Hero section is dark navy; Stitch uses a light surface with a subtle blueprint dot pattern
- Services section is a plain 2×2 card grid; Stitch uses a bento grid with background photos
- Thought Leadership uses numbered card chips; Stitch uses a sticky 2-column editorial layout
- Footer is 3 columns and missing a brand column; Stitch has 4 columns with red section headings
- Material Symbols icons are absent throughout

**Navigation wiring rule (strictly observed across all components):**
This is a multi-page static site. Only the home page is built. All nav bar links, footer page links, and CTA buttons ("Consult Experts" / "Consult Our Experts", "View Portfolio") must use `href="/404"` — they are not yet wired. The logo links to `/`. No in-page anchor links are used — Astro serves its own default 404 page for any unbuilt route.

## Step 1 — Hand-write UI primitives (`src/components/ui/`)

No shadcn/ui. No new dependencies. Use `clsx` + `tailwind-merge` (already installed) for the `cn()` helper.

Create `src/lib/utils.ts`:
```ts
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';
export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

Then hand-write these three components using the project's existing design tokens:

**`src/components/ui/Button.astro`** — used in: nav CTA, hero CTAs, footer newsletter submit.
Variants: `primary` (gradient fill, white text, xl radius) and `ghost` (transparent, outline-variant border, primary text).

**`src/components/ui/Badge.astro`** — used in: service card tags (NVH Simulation, Industry 4.0), hero overline chip.
Variants: `default` (secondary-fixed bg, on-secondary-fixed-variant text) and `green` (tertiary-fixed bg, on-tertiary-fixed-variant text).

**`src/components/ui/Input.astro`** — used in: footer newsletter email field.
Style: surface-container-lowest bg, no full border, 2px bottom accent in outline-variant → primary on focus.

These are `.astro` components (not React), since none require client-side interactivity.

**`src/pages/index.astro` meta:**
```astro
<BaseLayout

  description="Indvaltech delivers precision engineering services across aerospace, automotive, and industrial sectors — simulation, structural analysis, and manufacturing intelligence for demanding environments."
```
